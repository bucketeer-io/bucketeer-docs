Buckteersには、それぞれ異なるアクセスレベルを持つ2種類のアカウントがあります。

- **管理者アカウント**: 管理者アカウントは、すべての既存プロジェクトに完全なアクセスと制御を提供します。管理者アカウントを持つメンバーは、すべての既存環境にアクセスし、他のメンバーのアカウントを管理できます。

- **環境アカウント**: 環境アカウントは、プラットフォーム上の特定の環境に関連付けられています。環境アカウントを持つメンバーは、その特定の環境のすべての機能にアクセスできます。

:::note

オープンソースプロジェクトとして、Buckteersシステムバージョンをホストすることができます。この場合、独自の管理者アカウントを作成することになります。

Bucketeerがホストするシステムを使用している場合は、管理者アカウントを作成するにはBucketeerチームに連絡する必要があります。

:::

## 環境アカウントのロール

環境アカウントは、特定の環境に限定されていることに加えて、アカウント作成プロセス中に事前定義されたロールが割り当てられます。定義されたロールは、アカウントのアクセスレベルを変更します。環境アカウントで使用できるロールは次のとおりです。

- **閲覧者**: 閲覧者は、割り当てられた環境への読み取り専用アクセス権を持っています。すべてのデータを表示できますが、変更はできません。このロールは、フィーチャーフラグの可視性が必要でありながら、変更する権限は必要のない組織のメンバーに適しています。

- **編集者**: 編集者は、割り当てられた環境内のフィーチャーフラグ、ゴール、エクスペリメントなどを変更できます。ただし、新しいチームメンバーを追加したり、APIキーを作成したりすることはできません。

- **オーナー**: オーナーは、割り当てられた環境内で何でもすることができます。これにはチームメンバーやAPIキーの追加を含みます。
