スケジュールソリューションは、特定の日時に機能を提供または削除するように設計されています。 したがって、休日や特別なプロモーションなどの新しい機能をリリースするために使用できます。 同じソリューションを使用して、休暇後に一時的な機能を削除したり、一定期間後にベータ機能をオフにしたりできます。

## スケジュールの仕組み

スケジュール条件付きでBucketeerの自動操作を使用する場合、指定した日時に基づいてフラグを管理します。 Bucketeer が SDK リクエストを受け取ると、スケジュール条件は指定された日時に達したかどうかを評価します。 その場合、フラグのバリエーションを有効または無効にする操作が実行されます。 

:::info 要件と制限をスケジュールする

スケジュールの条件を使用するには、常に将来の日付と時刻を使用して操作をスケジュールする必要があります。  

スケジュール条件に基づいたアクティブなオペレーションは 2 つのみです。 1 つはフラグを有効にするタイミングを定義し、もう 1 つはフラグを無効にするタイミングを定義します。 ただし、両方の操作を使用する必要はありません。 たとえば、有効にするオプションのみを使用できます。

:::

## スケジュールによる自動操作の設定

スケジュールオプションを使用して機能フラグを解除または解除するには、以下の手順に従う必要があります。

1.  **自動操作** ページに移動します。
2.  "参加"ボタンを押下します。
3.  操作設定パネルが表示されます。 **有効化**を選択してリリースします。またはフラグを削除するには**キルスイッチ**を選択してください。
4.  **スケジュール** を選択します。
5.  日付と時刻を設定します。 将来の日付と時刻を選択する必要があることを覚えておいてください。
6.  **提出**をクリックします。
