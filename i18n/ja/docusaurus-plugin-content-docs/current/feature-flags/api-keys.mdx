このページでは、APIキーとは何か、APIキーの作成方法、BucketeerシステムにおけるAPIキーの役割について説明しています。

## APIキーとは

Bucketeerでフラグを使用する場合、APIキーは必要不可欠です。 Apiキーは、フィーチャーフラグ管理システムへのアクセスを認証および制御し、
アプリケーションとBucketeer管理サービス間のセキュアな接続を保証する認証機能を提供します。 各APIキーは特定のプロジェクトと環境に関連
付けられているため、アプリケーションを目的のユーザーグループに識別および関連付けることができます。 フラグ情報にアクセスするには有効な
APIキーが必要であるため、システムのセキュリティと整合性が維持され、不正なアクセスやフィーチャーフラグ設定の操作のリスクが軽減されます。

アプリケーションでSDKを設定するには、有効なAPIキーを含める必要があります。 このキーは、SDKがサーバーリクエストを実行する際に、サーバ
ーがリクエストが有効であることを認識するために必要です。 設定中に無効なAPIキーを提供すると、サーバーはリクエストを拒否します。 SDKの構
成の詳細については、[SDKのドキュメンテーション](../sdk) をご参照ください。

## APIキーの作成

APIキーを作成するには、管理者であるか、オーナーの役割を持つ環境アカウントが必要です。 他のメンバーはAPIキーを表示できますが、管理することはできません。

Bucketeer ダッシュボード上で新しい API キーを生成するには、**APIキー**タブに移動します。 このセクションには、現在使用可能なすべての API キーのリストが表示されます。 新しい Api キーを作成するには、**+ 追加**ボタンをクリックします。 画面の右側にサイドパネルが表示さ れます。 新しいキーの名前を入力し（最大100文字）、その後、**送信**をクリックします。 新しいキーは、**APIキー**リストの一番上に表示されます。

Bucketeer チームは、各API キーに関連付けられた機能またはプロジェクトに基づいて分かりやすい名前を付けることを推奨しています。 これにより、管理が容易になります。ランダムに生成された名前を使用すると混乱を招き、API キーがどこで使用されているのかを識別す ることが難しくなるため避けてください。

## APIキーの管理

API キーが作成されると、すぐに使用可能になり、**ON **状態に設定されます。 また、現在の環境内のすべてのフィーチャーフラグへのア
クセスが許可されます。 Bucketeer ダッシュボードには、API キーを削除するための直接的なオプションはありません。 ただし、APIキーが
不要になった場合は、**APIキー**タブにあるスイッチボタンを使用して状態を**OFF**に切り替えることができます。 このアクションにより、その特定のキーを使用したすべての SDK リクエストが拒否されます。
